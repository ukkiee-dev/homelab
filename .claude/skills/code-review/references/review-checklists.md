## 리뷰어별 빠른 체크리스트

각 리뷰어가 최소한 확인해야 하는 항목. 에이전트 정의에 상세 설명이 있고, 여기는 빠른 참조용.

### 아키텍처
- [ ] 모듈 경계가 명확한가 (내부 구현 노출 여부)
- [ ] 순환 의존 없는가
- [ ] 의존성 방향이 올바른가 (상위→하위)
- [ ] 불필요한 추상화가 없는가
- [ ] 에러 전파 전략이 일관적인가
- [ ] API 표면이 최소한인가

### 보안
- [ ] 사용자 입력이 sanitize 되는가
- [ ] SQL/XSS/Command 인젝션 벡터가 없는가
- [ ] 인증/인가 검사가 모든 엔드포인트에 적용되는가
- [ ] 하드코딩된 시크릿이 없는가
- [ ] 민감 데이터가 로그에 출력되지 않는가
- [ ] CORS, Rate limit이 설정되어 있는가

### 성능
- [ ] O(n²) 이상의 루프가 없는가
- [ ] N+1 쿼리가 없는가
- [ ] 대용량 데이터를 전체 로드하지 않는가
- [ ] 병렬화 가능한 직렬 호출이 없는가
- [ ] 이벤트 리스너/구독 해제가 되는가
- [ ] 불필요한 리렌더링이 없는가 (프론트엔드)

### 스타일
- [ ] 함수 길이가 적절한가 (30줄 이내 권장)
- [ ] 중첩 깊이가 3단계 이내인가
- [ ] 변수/함수명이 의도를 드러내는가
- [ ] 코드 중복이 3회 미만인가
- [ ] 데드 코드가 없는가
- [ ] 핵심 로직에 테스트가 있는가

## 교차 관심사 패턴

여러 리뷰 차원이 겹치는 영역. 오케스트레이터가 통합 시 이 패턴을 참고하여 교차 이슈를 식별한다.

| 교차 패턴 | 관련 차원 | 예시 |
|----------|----------|------|
| **에러 처리 공백 → 보안 취약점** | 스타일 + 보안 | catch 블록이 비어있어 인증 실패를 무시 |
| **God 클래스 → 보안 표면 증가** | 아키텍처 + 보안 | 한 클래스가 인증+데이터+렌더링을 모두 담당 → 권한 분리 불가 |
| **N+1 → 아키텍처 변경 필요** | 성능 + 아키텍처 | 쿼리 최적화로 불충분 → 캐싱 레이어 또는 데이터 로더 도입 필요 |
| **타입 불안전 → 런타임 오류** | 스타일 + 성능 | any 타입이 에러 핸들러에서 사용 → 프로덕션 crash |
| **코드 중복 → 불일치 위험** | 스타일 + 보안 | 인증 로직이 3곳에 복사됨 → 한 곳만 패치되면 나머지가 취약 |
| **과도한 추상화 → 성능 오버헤드** | 아키텍처 + 성능 | 불필요한 프록시/데코레이터 체인이 핫 패스에 위치 |

교차 관심사가 발견되면 심각도를 **한 단계 상향**한다 (Info→Warning, Warning→Critical).

## 통합 보고서 템플릿

```markdown
# 종합 코드 리뷰 보고서

**리뷰일**: YYYY-MM-DD
**대상**: [파일 목록 / PR / 브랜치]
**언어/프레임워크**: [감지된 스택]
**리뷰어**: 아키텍처, 보안, 성능, 스타일

---

## 요약

| 심각도 | 건수 |
|--------|------|
| Critical | N |
| Warning | N |
| Info | N |
| **합계** | **N** |

**즉시 조치 필요**: [Critical 항목 한 줄 요약]
**교차 관심사**: N건 발견

## Critical

| # | 파일:라인 | 차원 | 이슈 | 수정 제안 |
|---|---------|------|------|----------|
| 1 | `path:line` | 🔒보안 | [설명] | [코드 제안] |
| 2 | `path:line` | ⚡성능+🏗아키 | [교차 이슈] | [제안] |

## Warning

| # | 파일:라인 | 차원 | 이슈 | 수정 제안 |
|---|---------|------|------|----------|
| ... |

## Info

| # | 파일:라인 | 차원 | 이슈 | 수정 제안 |
|---|---------|------|------|----------|
| ... |

## 교차 관심사 분석

### [패턴명]
- **관련 차원**: [아키텍처 + 보안 등]
- **발견 위치**: [파일:라인]
- **설명**: [왜 이것이 교차 관심사인가]
- **통합 제안**: [양쪽 차원을 고려한 해결 방안]

## 차원별 상세

### 🏗 아키텍처
[arch-reviewer 핵심 발견 요약]

### 🔒 보안
[security-reviewer 핵심 발견 요약]

### ⚡ 성능
[perf-reviewer 핵심 발견 요약]

### 🎨 스타일
[style-reviewer 핵심 발견 요약 + 잘 된 점]

## 우선순위 액션 아이템

1. [ ] [가장 시급한 수정]
2. [ ] [두 번째]
3. [ ] ...

---
*이 보고서는 4개 전문 에이전트(아키텍처/보안/성능/스타일)의 독립 감사와 교차 분석을 거쳐 작성되었습니다.*
```
